# Meeting Notes Skill

将零散的会议记录整理成结构化的会议纪要。

## 触发条件

当用户提到以下关键词时加载此 Skill：
- 会议总结
- 会议纪要
- 整理会议
- 会议记录

## 工作流程

1. **识别会议内容**：如果用户没有提供原始会议记录，主动询问用户提供
2. **信息提取**：从用户提供的内容中提取关键信息
3. **结构化整理**：按照输出格式组织信息
4. **行动项标注**：特别注意提取带明确责任人和截止时间的行动项
5. **输出纪要**：生成结构化的会议纪要

## 输出格式

```
# [会议主题]

**时间**：YYYY-MM-DD
**参会人员**：[人员列表]

## 讨论内容

1. [第一个议题]
   - 要点1
   - 要点2

2. [第二个议题]
   - 要点1
   - 要点2

## 行动项

| 行动项 | 负责人 | 截止时间 |
|--------|--------|----------|
| [具体任务] | [姓名] | [日期] |
| [具体任务] | [姓名] | [日期] |

## 待跟进事项

- [其他需要关注的问题]
```

## 处理原则

1. 如果会议记录中没有明确的信息字段（如时间、参会人），在输出中标注 `[待补充]`
2. 行动项必须包含**明确的行动内容 + 负责人 + 截止时间**，三者缺一不可
3. 如果原始内容包含多个议题，按逻辑顺序组织，而非按出现顺序
4. 讨论内容要精简，避免直接复制粘贴原始记录

## 示例

### 输入（用户提供的原始记录）
```
今天和张三、李四开会讨论Q2产品规划
张三说市场反馈不错，建议加一个聊天功能
李四担心技术实现太复杂
我们决定先做MVP版本，3周内完成
张三负责产品设计，下周一之前
李四负责技术方案，3天后输出
```

### 输出
```
# Q2 产品规划讨论

**时间**：[待补充]
**参会人员**：张三、李四

## 讨论内容

1. 市场反馈与功能规划
   - 市场反馈良好
   - 建议新增聊天功能

2. 技术可行性评估
   - 担心实现复杂度高
   - 决定先做 MVP 版本

## 行动项

| 行动项 | 负责人 | 截止时间 |
|--------|--------|----------|
| 完成聊天功能产品设计 | 张三 | 下周一 |
| 输出技术方案评估 | 李四 | 3天后 |

## 待跟进事项

- MVP 版本计划 3 周内完成
```
